# 13. Risks and Mitigations

## Purpose
This document identifies key project and operational risks along with
mitigation approaches.

## Identified Risks

| Risk | Description | Mitigation |
|-----|------------|------------|
| Data inconsistency | Manual data entry errors | Validation rules, controlled entry points |
| Missed appointments | Schedule conflicts | Master agreement before creation, calendar sync |
| Financial mismatch | Price differs from expectation | Price fixed only at payment stage |
| Dependency on admin | Single operational role | Simple flows, minimal configuration |

## Notes
Risks are reviewed as the system evolves.
